# 인증 기능 설정 가이드

## 개요
회원가입 및 로그인 기능이 추가되었습니다. 이메일/비밀번호 기반 인증과 JWT 토큰을 사용합니다.

## 백엔드 설정

### 1. 환경 변수 설정

`server/.env` 파일에 다음 환경 변수를 추가하세요:

```env
# JWT
JWT_SECRET="your-secret-key-change-in-production"
JWT_EXPIRES_IN="15m"
JWT_REFRESH_EXPIRES_IN="7d"
```

**중요**: 프로덕션 환경에서는 `JWT_SECRET`을 강력한 랜덤 문자열로 변경하세요.

### 2. 데이터베이스 마이그레이션

Prisma 스키마가 변경되었으므로 마이그레이션을 실행해야 합니다:

```bash
cd server
npm run prisma:migrate
```

이 명령은 다음을 수행합니다:
- `User` 테이블 생성
- `Project` 테이블에 `userId` 컬럼 추가 (기존 데이터는 null 허용)

### 3. 서버 재시작

마이그레이션 후 서버를 재시작하세요:

```bash
npm run dev
```

## 프론트엔드 설정

### 1. 백엔드 서버 주소 설정

프론트엔드에서 설정(⚙️) 버튼을 클릭하고 백엔드 서버 주소를 입력하세요:
- 예: `http://localhost:3001`

### 2. 회원가입 및 로그인

1. `/register` 페이지에서 회원가입
2. `/login` 페이지에서 로그인
3. 로그인 후 자동으로 홈(`/`)으로 리다이렉트됩니다

## API 엔드포인트

### 인증 (공개)
- `POST /api/auth/register` - 회원가입
- `POST /api/auth/login` - 로그인
- `POST /api/auth/logout` - 로그아웃

### 인증 필요
- `GET /api/auth/me` - 현재 사용자 정보
- `GET /api/projects` - 프로젝트 목록 (사용자별)
- `POST /api/projects` - 프로젝트 생성 (자동으로 userId 설정)
- 기타 모든 프로젝트/스냅샷/Diff API

## 보안 기능

1. **비밀번호 해싱**: bcrypt (cost factor 12)
2. **JWT 토큰**: Access Token (15분), Refresh Token (7일)
3. **자동 로그아웃**: 401 응답 시 자동으로 로그아웃 처리
4. **권한 확인**: 사용자는 자신의 프로젝트만 접근 가능

## 사용 방법

1. **회원가입**
   - 이메일, 비밀번호(최소 8자, 영문+숫자), 이름(선택) 입력
   - 회원가입 성공 시 자동 로그인

2. **로그인**
   - 이메일과 비밀번호로 로그인
   - JWT 토큰이 localStorage에 저장됨

3. **프로젝트 관리**
   - 로그인 후 생성한 프로젝트는 자동으로 사용자에게 연결됨
   - 다른 사용자의 프로젝트는 접근 불가

4. **로그아웃**
   - Dashboard에서 "로그아웃" 버튼 클릭
   - 또는 토큰 만료 시 자동 로그아웃

## 문제 해결

### "인증이 필요합니다" 에러
- 백엔드 서버가 실행 중인지 확인
- 프론트엔드 설정에서 백엔드 서버 주소가 올바른지 확인
- 브라우저 개발자 도구에서 네트워크 탭 확인

### "이미 사용 중인 이메일입니다" 에러
- 다른 이메일로 회원가입 시도
- 또는 로그인 페이지에서 로그인

### 마이그레이션 실패
- PostgreSQL이 실행 중인지 확인: `lsof -ti:5432`
- 데이터베이스가 존재하는지 확인: `psql -l | grep api_watcher`
- 기존 데이터가 있으면 백업 후 마이그레이션 실행
